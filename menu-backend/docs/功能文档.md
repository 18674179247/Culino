# Menu Backend 功能文档

基于 Rust (Axum + SQLx + Redis) 的菜谱管理后端服务，面向家庭烹饪爱好者，提供菜谱发现、烹饪管理、AI 辅助等功能。

## 使用场景

面向 C 端用户（家庭烹饪爱好者）的移动端/Web 菜谱平台后端。用户可以浏览菜谱、收藏、记录烹饪、规划膳食，并通过 AI 获取营养分析和个性化推荐。

---

## 一、用户系统

| 功能 | 方法 | 路径 | 展示内容 |
|------|------|------|----------|
| 注册 | POST | `/api/v1/user/register` | JWT Token + 用户信息（ID、用户名、昵称、头像、角色） |
| 登录 | POST | `/api/v1/user/login` | 同上 |
| 个人信息 | GET | `/api/v1/user/me` | 用户 ID、用户名、昵称、头像、角色、注册时间 |
| 编辑资料 | PUT | `/api/v1/user/me` | 更新后的昵称、头像 |
| 登出 | POST | `/api/v1/user/logout` | 撤销当前 Token |

- 密码使用 Argon2 哈希，JWT 有效期 7 天
- Token 存储于 Redis，登出即撤销
- 支持 admin/user 角色

## 三、菜谱

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建菜谱 | POST | `/api/v1/recipe` | 需登录 |
| 菜谱详情 | GET | `/api/v1/recipe/{id}` | 公开访问 |
| 更新菜谱 | PUT | `/api/v1/recipe/{id}` | 仅作者 |
| 删除菜谱 | DELETE | `/api/v1/recipe/{id}` | 仅作者 |
| 搜索菜谱 | GET | `/api/v1/recipe/search` | 公开访问 |
| 随机推荐 | GET | `/api/v1/recipe/random` | 公开访问 |

### 菜谱详情包含

- **基本信息**：标题、描述、封面图、难度(1-5)、烹饪时间、准备时间、份数、来源
- **食材清单**：名称、用量、单位、备注
- **调料清单**：名称、用量、单位
- **步骤列表**：序号、内容、图片、每步时长
- **标签**：如"川菜""低脂""快手菜"
- **营养信息**（AI 分析后展示）：热量、蛋白质、脂肪、碳水、纤维、钠、健康评分、健康标签、适合人群

### 搜索支持

- 关键词（标题/描述模糊匹配）
- 难度筛选
- 最大烹饪时间
- 标签 ID（逗号分隔，AND 逻辑）
- 食材 ID（逗号分隔，AND 逻辑）
- 分页（默认每页 20 条，最大 100）

## 七、AI 智能功能

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 营养分析 | POST | `/api/v1/ai/nutrition/analyze/{recipe_id}` | 调用 DeepSeek API |
| 查询营养 | GET | `/api/v1/ai/nutrition/{recipe_id}` | 读取缓存结果 |
| 个性化推荐 | GET | `/api/v1/ai/recommend/personalized` | 需登录 |
| 相似菜谱 | GET | `/api/v1/ai/recommend/similar/{recipe_id}` | |
| 热门推荐 | GET | `/api/v1/ai/recommend/trending` | |
| 健康推荐 | GET | `/api/v1/ai/recommend/health/{goal}` | 需登录 |
| 偏好分析 | POST | `/api/v1/ai/preference/analyze` | 需登录 |
| 偏好画像 | GET | `/api/v1/ai/preference/profile` | 需登录 |
| 行为日志 | POST | `/api/v1/ai/behavior/log` | 需登录 |

### 营养分析展示

调用 DeepSeek AI 分析菜谱后返回：
- **营养成分**：每份热量(kcal)、蛋白质(g)、脂肪(g)、碳水(g)、纤维(g)、钠(mg)
- **健康评分**：1-100 分
- **健康标签**：如"低脂""高蛋白""低钠""高纤维"
- **适合人群**：如"减脂人群""健身人群""儿童""老人""孕妇"
- **注意事项**：如"高盐""高糖""高胆固醇"
- **分析说明**：100-200 字营养解读

### 推荐系统

| 推荐类型 | 算法逻辑 |
|----------|----------|
| 个性化推荐 | 基于用户收藏标签分布 → 匹配相似标签菜谱 → 排除已收藏 → 按难度/时间匹配度打分 |
| 相似菜谱 | 基于标签重合度 → 计算相似度分数 → 按共同标签数排序 |
| 热门推荐 | 收藏数(0.7) + 评分(0.3) + 30 天时间衰减(2.0) → 新菜谱也有曝光机会 |
| 健康推荐 | 按健康目标(减脂/增肌/保持健康)匹配营养标签 → 筛选健康评分 ≥70 的菜谱 |

### 用户偏好画像

基于收藏 + 烹饪记录 + 评分数据，自动分析：
- **菜系偏好**（如川菜 45%、粤菜 30%）
- **口味偏好**（如麻辣 40%、清淡 30%）
- **食材偏好**（Top 20 常用食材及权重）
- **标签偏好**（综合权重分布）
- **平均烹饪时间**、**难度偏好**
- **总收藏数**、**总烹饪次数**、**平均评分**

## 技术栈

| 组件 | 技术选型 |
|------|----------|
| 框架 | Axum 0.8 |
| 数据库 | PostgreSQL 16 + SQLx 0.8 |
| 缓存 | Redis 7 |
| 对象存储 | S3 兼容（MinIO / RustFS / 阿里云 OSS） |
| 认证 | JWT + Argon2 |
| AI | DeepSeek API (deepseek-chat) |
| 文档 | utoipa + Swagger UI |
| 部署 | Docker Compose |
| CI | GitHub Actions (fmt + clippy + test + build) |
