# 菜谱应用前端模块说明

## 已完成模块

### 1. 菜谱模块 (feature:recipe)
**功能:**
- 菜谱列表浏览（支持分页）
- 菜谱搜索（关键词、难度筛选）
- 菜谱详情查看（包含食材、调料、步骤、营养信息）
- 随机推荐菜谱
- 创建/编辑菜谱（待实现UI）

**核心文件:**
- `RecipeListScreen.kt` - 菜谱列表界面
- `RecipeDetailScreen.kt` - 菜谱详情界面
- `RecipeApi.kt` - API接口定义
- `RecipeRepository.kt` - 数据仓库

**对应后端接口:**
- `GET /api/v1/recipe/search` - 搜索菜谱
- `GET /api/v1/recipe/{id}` - 获取详情
- `GET /api/v1/recipe/random` - 随机推荐
- `POST /api/v1/recipe` - 创建菜谱
- `PUT /api/v1/recipe/{id}` - 更新菜谱
- `DELETE /api/v1/recipe/{id}` - 删除菜谱

### 2. 社交模块 (feature:social)
**功能:**
- 收藏菜谱管理
- 烹饪记录管理（评分、备注）

**核心文件:**
- `SocialApi.kt` - API接口定义
- `SocialRepository.kt` - 数据仓库
- `SocialModels.kt` - 数据模型

**对应后端接口:**
- `GET /api/v1/social/favorites` - 获取收藏列表
- `POST /api/v1/social/favorites/{recipe_id}` - 添加收藏
- `DELETE /api/v1/social/favorites/{recipe_id}` - 取消收藏
- `GET /api/v1/social/cooking-logs` - 获取烹饪记录
- `POST /api/v1/social/cooking-logs` - 创建烹饪记录
- `PUT /api/v1/social/cooking-logs/{id}` - 更新记录
- `DELETE /api/v1/social/cooking-logs/{id}` - 删除记录

### 3. 食材模块 (feature:ingredient)
**功能:**
- 食材列表浏览
- 食材分类查看
- 食材详情

**对应后端接口:**
- `GET /api/v1/ingredient/ingredients` - 获取食材列表
- `GET /api/v1/ingredient/ingredients/{id}` - 获取食材详情
- `GET /api/v1/ingredient/ingredient-categories` - 获取分类

### 4. 工具模块 (feature:tool)
**功能:**
- 购物清单管理
- 膳食计划管理

**对应后端接口:**
- `GET /api/v1/tool/shopping-lists` - 获取购物清单
- `POST /api/v1/tool/shopping-lists` - 创建购物清单
- `GET /api/v1/tool/shopping-lists/{id}` - 获取清单详情
- `POST /api/v1/tool/shopping-lists/{id}/items` - 添加商品
- `PUT /api/v1/tool/shopping-lists/{list_id}/items/{item_id}` - 更新商品
- `DELETE /api/v1/tool/shopping-lists/{list_id}/items/{item_id}` - 删除商品
- `GET /api/v1/tool/meal-plans` - 获取膳食计划
- `POST /api/v1/tool/meal-plans` - 创建膳食计划
- `PUT /api/v1/tool/meal-plans/{id}` - 更新计划
- `DELETE /api/v1/tool/meal-plans/{id}` - 删除计划

### 5. AI模块 (feature:ai)
**功能:**
- 营养分析
- 个性化推荐
- 相似菜谱推荐
- 热门推荐
- 健康目标推荐
- 用户偏好分析

**对应后端接口:**
- `POST /api/v1/ai/nutrition/analyze/{recipe_id}` - 分析营养
- `GET /api/v1/ai/nutrition/{recipe_id}` - 获取营养信息
- `GET /api/v1/ai/recommend/personalized` - 个性化推荐
- `GET /api/v1/ai/recommend/similar/{recipe_id}` - 相似推荐
- `GET /api/v1/ai/recommend/trending` - 热门推荐
- `GET /api/v1/ai/recommend/health/{goal}` - 健康目标推荐
- `POST /api/v1/ai/preference/analyze` - 分析偏好
- `GET /api/v1/ai/preference/profile` - 获取偏好画像
- `POST /api/v1/ai/behavior/log` - 记录行为

## 项目结构

```
culino-mobile/
├── feature/
│   ├── user/          # 用户模块（已存在）
│   ├── recipe/        # 菜谱模块（新增）
│   ├── social/        # 社交模块（新增）
│   ├── ingredient/    # 食材模块（新增）
│   ├── tool/          # 工具模块（新增）
│   └── ai/            # AI模块（新增）
├── core/
│   ├── common/        # 通用工具
│   ├── network/       # 网络层
│   └── ui/            # UI组件
└── composeApp/        # 主应用
```

## 架构说明

每个feature模块采用Clean Architecture分层:

```
feature/
└── src/commonMain/kotlin/com/culino/feature/{module}/
    ├── data/              # 数据层
    │   ├── *Models.kt     # 数据模型
    │   ├── *Api.kt        # API接口
    │   └── *Repository.kt # 数据仓库
    ├── domain/            # 业务逻辑层
    │   └── *UseCases.kt   # 用例
    └── presentation/      # 表现层
        ├── *Screen.kt     # UI界面
        ├── *ViewModel.kt  # 视图模型
        └── *State.kt      # UI状态
```
